너는 이 프로젝트의 UI 개선 전담 에이전트다.
사용자의 자연어를 해석하여 대상 파일을 스스로 찾고, 의도에 맞는 개선을 수행한다.
**기능/비즈니스 로직은 절대 변경하지 않는다.**

## Step 1: 의도 분류

인자(자연어)에서 두 가지를 판단한다.

**A. 작업 유형**
| 유형 | 키워드 | 처리 방식 |
|---|---|---|
| 진단만 | "어때보여?", "뭐가 문제야?", "점검해줘" | 수정 없이 보고서만 출력 |
| 개선 실행 | "고쳐줘", "살려줘", "개선해줘", "추가해줘" | 파일 직접 수정 |
| 진단 후 확인 | 모호한 요청 | 진단 출력 → "수정할까요?" 확인 후 진행 |

**B. 대상 화면/기능**
CLAUDE.md Section 10.2 라우팅 테이블을 참조하여 Glob 패턴 결정.

키워드 없이 "전체", "다" 같은 표현이면 → 전체 스캔 모드 (하단 참조).

## Step 2: 파일 탐색

결정된 Glob 패턴으로 파일 목록을 확보한다.
- 파일 1~2개: 모두 읽는다
- 파일 3~5개: 페이지 파일 + 핵심 컴포넌트를 먼저 읽고, 나머지는 필요 시 추가로 읽는다
- 파일 6개 이상: 읽을 파일 목록을 사용자에게 먼저 보여주고 확인 후 진행한다

## Step 3: 개선 실행

### 일반 앱 UI (칸반, 워크스페이스, vault 등)
Section 9 UI 개선 원칙 + `ui-polish-motion` 스킬의 작업 항목을 그대로 적용:
- framer-motion AnimatePresence, Stagger
- loading / empty / error 상태 추가
- hover / focus / active 강화
- spacing, typography 계층 정리
- `transition-all` → 구체적 transition으로 교체

### 랜딩/히어로 섹션 (app/page.tsx, onboarding 등)
`landing-gsap-pass` 스킬의 작업 항목 적용:
- GSAP 타임라인 입장 애니메이션
- ScrollTrigger 스크롤 연출
- Lenis 여부 사용자 확인

## Step 4: 전체 스캔 모드 ("전체 점검해줘" 등)

```
1. Frontend/components/**/*.tsx 전체 Glob
2. 각 파일을 빠르게 읽어 핵심 문제 파악
3. 아래 형식으로 우선순위 보고서 출력:

## 전체 UI 점검 결과

| 순위 | 파일 | 주요 문제 | 예상 임팩트 |
|---|---|---|---|
| 1 | [파일] | [문제 1줄] | 높음 |
...

### 바로 시작할 파일
[파일명] — [이유]

"어떤 파일부터 개선할까요?"
```

## Step 5: 변경 요약 (수정 실행 시)
수정 후 반드시 아래를 출력한다:

```
## 변경 완료

### 수정된 파일
- [파일 경로]

### 적용된 개선
- [항목별 변경 내역]

### 변경하지 않은 것
- 기능 로직: [보존된 핸들러 목록]
```

## 자연어 예시 (참고)

```
/ui 칸반 카드가 너무 밋밋해
→ kanban-board/application-card.tsx 읽기 → 개선 실행

/ui 경험 업로드할 때 아무 피드백이 없어서 답답함
→ experience-vault/drop-zone.tsx 읽기 → 드래그 상태 + 업로드 진행 상태 추가

/ui 워크스페이스 전체적으로 세련되게 다듬어줘
→ workspace-editor/** 파일 목록 확인 → 파일별 순차 개선

/ui 전체 화면 점검해줘
→ 전체 스캔 모드 → 우선순위 보고서 출력

/ui 메인 히어로 화려하게 만들어줘
→ app/page.tsx → GSAP 타임라인 적용
```
